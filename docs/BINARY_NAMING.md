# Binary Naming

Beginning with the N-API migration, binary artifacts follow the standard
prebuild naming convention.

Examples

```
electron-printer-v3.1.0-napi-v8-win32-x64.tar.gz

electron-printer-v3.1.0-napi-v8-win32-ia32.tar.gz

electron-printer-v3.1.0-napi-v8-linux-x64.tar.gz

electron-printer-v3.1.0-napi-v8-darwin-arm64.tar.gz
```

These binaries are runtime independent and may be shared across supported
Node.js and Electron releases.
